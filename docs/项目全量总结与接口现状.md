> 生成时间：2026-03-23  
> 说明：本文件基于当前仓库源码实际状态整理（前端小程序 + 后端 NestJS）。

## 1. 项目整体做了什么

### 1.1 架构与技术

- 前端：微信小程序（页面 + `services` 统一请求层）
- 后端：NestJS + Prisma + MySQL + Redis + JWT + Swagger
- 统一响应：`{ code, message, data }`
- 请求封装能力已具备：鉴权头注入、错误 toast、401 处理、上传封装、mock 降级

### 1.2 主要业务模块（前后端都已具备）

- 认证登录：登录、发送验证码、获取个人信息、退出
- 批次管理：列表、详情、创建、阶段更新
- 农事日志：列表、创建、类型字典
- 病害识别：上传图片识别、识别记录查询
- 风险预警：摘要、评估、历史
- 定价：分级定价、基准价、建议
- 产品：列表、二维码生成
- 溯源：查询溯源聚合信息、校验链路
- 天气：实时天气、天气预报
- 上传：图片上传

## 2. 还有什么没有做（按源码现状）

## 2.1 明确未完成/占位项

- `pages/product-qrcode/index` 的“分享”仍是占位提示（`分享功能开发中`）

- 一些前端调用路径与后端当前控制器路径不一致（详见第 5 节“差异清单”）
- 参数命名存在蛇形/驼峰不统一（如 `batch_id` vs `batchId`，`pageSize` vs `limit`）

## 3. 前端 Service 接口总表（含 mock/真实策略）

说明：

- “真实调用”指 `request`/`uploadFile` 发起的真实 HTTP 路由
- “mock策略”分三类：
  - `无mock`：只走真实接口
  - `可降级mock`：`tryReal(real, mock)`，失败可降级
  - `强制mock（条件）`：在特定输入下直接走 mock
- 全局开关在 `services/config.js`：
  - `USE_MOCK=true`：全部走 mock
  - `USE_MOCK=false` + `MOCK_FALLBACK=true`：真实优先，失败降级 mock

## 5. 前后端差异清单（需要优先收敛）

以下为“前端当前调用”与“后端当前路由”不一致项：

1. 病害识别上传

- 前端：`POST /ai/disease/predict`（`batch_id`）  
- 后端：`POST /api/disease/predict`（DTO 字段 `batchId`）

1. 果实分级

- 前端：`POST /ai/fruit/grade`（图片上传）  
- 后端：`POST /api/price/grade`（规则定价 DTO，非图片上传）

1. 定价建议

- 前端：`GET /api/price/suggestion/:batchId`  
- 后端：`GET /api/price/suggestion?batchId=...`

1. 二维码生成

- 前端：`POST /api/qrcode/generate`（传 `batch_id`）  
- 后端：`POST /api/products/:id/qrcode`（按产品 ID）

1. 溯源查询

- 前端：`GET /api/trace/:batchId`  
- 后端：`GET /api/trace/:code`（语义是 traceCode）

1. 链上校验

- 前端：`GET /chain/verify/:batchId`  
- 后端：`GET /api/trace/:code/verify`（当前无 `chain` 控制器）

1. 手动日志页图片上传

- 前端 `pages/farming-log-create/index.js`：`POST /api/upload`  
- 后端：`POST /api/upload/image`

## 6. Mock 与真实接口归类结论

### 6.1 前端层面

- 全部业务 service 都具备“真实请求能力”
- 除 `auth.login/getProfile/logout` 与 `batch.create` 外，大多数接口均支持 mock 降级
- 当前配置为：`USE_MOCK=false`、`MOCK_FALLBACK=true`（真实优先，失败自动 mock）

### 6.2 后端层面

- 大部分接口已“真实落库/真实业务逻辑”
- 仍含 mock/模拟实现的关键能力：
  - 病害识别：`mockRecognizeDisease`（规则+随机）
  - 天气服务：`MockWeatherProvider`
  - 验证码发送：未接第三方短信网关
  - 登出：未做 JWT 黑名单失效

## 7. 建议下一步（按优先级）

1. 先统一前后端接口契约（第 5 节 7 项）
2. 再关闭前端自动降级：联调稳定后将 `MOCK_FALLBACK=false`
3. 将病害识别接入真实 AI 服务（替换 `mockRecognizeDisease`）
4. 将天气 provider 切到真实天气 API（替换 `MockWeatherProvider`）
5. 补齐登出失效策略（Redis 黑名单/短 token + refresh token）
